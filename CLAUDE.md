# Правила проекта и архитектурный подход

## Цель проекта

Разработать простой, но полностью рабочий сайт-блог на:

- PHP 8.1+
- MySQL
- Smarty Template Engine
- Без использования фреймворков
- С поддержкой Docker

Проект должен :

- чистую структуру кода;
- писать PHP без фреймворков;
- работу с MySQL;
- архитектурное мышление;
- самостоятельную реализацию.

Главный принцип — **простота, читаемость и понятность решения**.


## Стиль кода

Использовать:

- PSR-12
- strict types
- typed properties
- понятные имена методов и переменных

Во всех PHP-файлах использовать:

```php
declare(strict_types=1);
```

Код должен быть максимально явным и читаемым.

Предпочитать читаемость вместо «умного» кода.

# Ответственность слоёв

## Controllers

Контроллер отвечает только за:

- получение запроса;
- вызов сервисов;
- передачу данных в шаблон.
- данные передать через DTO

Контроллер НЕ должен:

- содержать SQL;
- содержать сложную бизнес-логику;
- работать напрямую с PDO.

Контроллер должен быть «тонким».

Пример:

Плохо:

```php
$sql = "SELECT * FROM posts";
```

Хорошо:

```php
$posts = $this->postService->getLatestPosts();
```

---

## Repositories

Репозитории отвечают только за работу с БД.

Только здесь разрешено:

- SQL;
- PDO;
- SELECT/INSERT/UPDATE.

Правила:

- использовать Prepared Statements;
- избегать SQL в контроллерах;
- писать читаемые запросы.
- писать безопасный sql Запросы 
- должно быть защита от sql инекци

ORM использовать нельзя.

---

## Services

Сервисный слой содержит бизнес-логику.

Примеры:

- получение главной страницы;
- сортировка статей;
- похожие статьи;
- логика категорий.

Сервис может использовать несколько репозиториев.

---

## Models

Модели — простые DTO-объекты.

Не использовать Active Record.



---

## Views (Smarty)

Использовать только Smarty.

В шаблонах НЕ должно быть:

- SQL;
- бизнес-логики;
- сложных вычислений.

Допустима только простая логика отображения:

- циклы;
- if;
- вывод данных.

# SQL правила

Обязательно:

- Prepared Statements
- bindValue / execute params
- защита от SQL Injection

Нельзя:

Плохо:

```php
$sql = "SELECT * FROM posts WHERE id = $id";
```

Хорошо:

```php
$stmt = $pdo->prepare(
    'SELECT * FROM posts WHERE id = :id'
);
```
